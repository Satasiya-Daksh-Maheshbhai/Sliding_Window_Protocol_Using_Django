About This Simulator

This simulator models reliable data communication over an unreliable network using Sliding Window Protocols. It demonstrates how transport-layer protocols like TCP ensure correct packet delivery using acknowledgments, timeouts, retransmissions, and sequence numbers.

  • Stop-and-Wait: Sends one packet at a time and waits for acknowledgment. Simple but inefficient when delay is high.
  • Go-Back-N: Uses a sliding window and retransmits the entire window if a packet is lost. Improves utilization but may waste bandwidth.
  • Selective Repeat: Retransmits only the lost packets and buffers out-of-order packets. Provides highest efficiency in lossy networks.

What This Simulator Analyzes:

  • • Transmission Delay (Tt)
  • • Propagation Delay (Tp)
  • • Round Trip Time (RTT)
  • • Efficiency (η)
  • • Throughput
  • • Success Rate

Insight: As packet loss or propagation delay increases, Stop-and-Wait performance degrades significantly. Sliding Window protocols maintain higher throughput by utilizing bandwidth efficiently. Selective Repeat performs best under high-loss conditions.
